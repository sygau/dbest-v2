# JUPAS Bookmarks + Homepage

Shipped 2026-05-17.

## What

- Bookmark (star) JUPAS programmes from any programme card.
- `/jupas/bookmarks` — dedicated watchlist tab (noindex).
- `/jupas` — calculator-centric JUPAS homepage with SEO content.
- Sidebar link to the calculator under 工具.

## Why bookmark by code, not by result

A student's predicted grades change repeatedly. Saving a scored snapshot would
go stale. So bookmarks store only JS codes (`jupas:starred:v1`). The watchlist
re-scores whenever the user runs the calculator again.

## Data model — hybrid bookmark tab

The bookmark tab needs programme data for codes the user may never have searched.
Two sources:

- `public/jupas/programmes-lite.json` — static catalogue (cutoffs + programme
  info, no scoring internals). Generated by `JUPASTool/scripts/process-data.js`
  alongside `programmes.json`. Fetched at runtime — kept out of the JS bundle.
- `jupas:lastresult:v1` (localStorage) — the most recent calculation, written by
  the calculator on every successful search.

Each starred card shows static cutoff data always; if the last calculation
contains that programme, the user's score/tier is overlaid.

## Key files

- `lib/jupas/client/bookmarks.ts` — store + `useBookmarks()` hook. Same-tab sync
  via `CustomEvent('jupas:bookmarks')`, cross-tab via the native `storage` event.
- `lib/jupas/client/lastResult.ts` — last-calculation snapshot (24h TTL).
- `lib/jupas/client/cardProps.ts` — `resultToCardProps` / `programmeToCardProps`,
  shared by the calculator and bookmark tab.
- `components/jupas/parts.tsx` — `JupasCard` bookmark button (under the uni
  logo); `prob` is now optional — un-scored cards render a 未計算 chip.
- `pages/jupas/bookmarks.tsx`, `pages/jupas/index.tsx`.

## Regenerating the catalogue

`programmes-lite.json` is rebuilt whenever the data pipeline runs:

```
cd JUPASTool/scripts && node process-data.js
```

## Notes / follow-ups

- The lite JSON (245 KB) is committed under `public/`. If programme data
  changes, re-run the build script and commit the regenerated file.
- No pagination on the bookmark tab — fine for realistic watchlist sizes
  (tens of programmes); revisit if needed.
